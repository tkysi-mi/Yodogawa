# 非機能要件の標準ベースライン

SKILL.md 手順6で「標準的な設定で仮置き」する際の提案値。

## Web アプリケーションの一般的な目標値

### パフォーマンス

- ページ読み込み: 3 秒以内
- API 応答: 500ms 以内

### セキュリティ

- HTTPS 必須
- パスワードはハッシュ化（bcrypt 推奨）
- 認証方式: 基本認証 / OAuth

### 可用性

- 稼働率: 99.5%（平日日中）
- 日次バックアップ

### スケーラビリティ

- 想定同時接続数の 2 倍のバースト耐性
- 水平スケーリング可能なアーキテクチャ

### ユーザビリティ

- WCAG 2.1 レベル AA 準拠（アクセシビリティ）
- 対応ブラウザ: 最新 2 バージョン（Chrome / Safari / Edge / Firefox）
- レスポンシブデザイン（PC / タブレット / モバイル）

### 保守性

- 構造化ログ（JSON 形式）
- 監視: APM + アラート通知
- デプロイ: 日次 or オンデマンド
